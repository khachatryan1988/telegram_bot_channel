Domus Giveaway Bot Audit

Status: issues found

1. Active draw is never finalized after winner response.
Files:
- app/services/draw.py:30-33
- app/handlers/user.py:145-160
- app/repositories/winners.py:75-91

Problem:
When the winner confirms the response, the winner status becomes `responded`, but the related draw remains `active`.
As a result, `draw_winner()` will keep returning `None` because it blocks whenever any active draw exists.

Impact:
- the giveaway lifecycle has no clean terminal state after a successful response
- a future draw for the same bot instance cannot start without manual DB intervention

2. `update_winner_status()` clears timestamps unintentionally.
File:
- app/repositories/winners.py:32-49

Problem:
Every status update writes both `responded_at` and `expired_at`, even when the caller does not intend to change them.
This makes the function destructive by default and can erase previously saved timestamps on later status changes.

Impact:
- winner state history is fragile
- timestamps can be lost silently

3. Duplicate update protection is process-local only.
Files:
- app/middlewares/dedup.py:12-38
- main.py:24

Problem:
Deduplication is kept only in memory. After restart, the process has no knowledge of previously seen updates.

Impact:
- protection helps against duplicates during one process lifetime
- it does not provide durable duplicate protection across restarts or crash recovery

4. README states Python 3.14.x, but validation was performed on Python 3.12.
File:
- README.md:14-18

Problem:
The runtime and tests were executed under Python 3.12 in the current environment, while the README advertises Python 3.14.x.

Impact:
- onboarding expectations and actual tested runtime are inconsistent

5. Generated `__pycache__` artifacts are present in the project tree.
Examples:
- tests/__pycache__/
- app/**/__pycache__/

Problem:
Generated bytecode caches are present in the workspace tree and should usually be ignored rather than kept as project artifacts.

Impact:
- repository noise
- harder review diffs if version-controlled
